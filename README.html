Overview

What It Is

A self-contained vessel maintenance log that runs as a full-screen app on your iPad. All data is stored locally on the device — no internet connection required after the first load, no accounts, no sync. One HTML file does everything.

Setup

Installing on iPad

1
Get the file onto your iPad
AirDrop — Easiest
  1. Right-click MyDashboard.html on your Mac → Share → AirDrop
  2. Select your iPad
  3. Tap Accept — opens in Safari automatically
Email
  1. Email the file to yourself as an attachment
  2. Open Mail → tap attachment → Share → Open in Safari
iCloud Drive
  1. Copy file to iCloud Drive on your Mac
  2. iPad → Files app → iCloud Drive → tap the file
  3. Tap Share → Open in Safari
2
Add to Home Screen

With the file open in Safari, tap the Share button (box with arrow pointing up). Scroll down and tap Add to Home Screen. Name pre-fills as Sunset Serenade — edit if desired. Tap Add.

The app launches full-screen with no browser chrome, just like a native app.

💡
Tip: Portrait orientation works best. Tap Settings in the upper right to configure the app.
3
First Launch

On first open the app loads pre-populated seed data from the vessel's maintenance history. The seed writes once only — it will never overwrite your data.

Updates

Upgrading the App

When a new version of MyDashboard.html is available:

Export Backup first — Settings → Export Backup → save the .json file
Replace MyDashboard.html (AirDrop, email, or iCloud)
Open the new file in Safari → Add to Home Screen
Your data loads automatically from localStorage — nothing is overwritten
Confirm version in Settings footer
v1 to v2 migration: existing intervals stamped as "hours" type automatically

If the new version includes a data schema change, migrations run automatically before any data is touched. The version footer in Settings shows VERSION x.x.x · DATA vx.

⚠️
If something goes wrong: use Import Backup in Settings to restore from the .json file exported in step 1.

Tabs

Overview

📊
Overview
Main status screen — engine hours and service intervals at a glance

Engine Hours row — Port, Generator, and Stbd as large metric cards at the top. Derived from the highest hours in log entries, or overridden manually.

⏱ Update Engine Hours — enter current hours for any system without creating a log entry. Manual values override log-derived values for interval calculations.

Collapsible sections — each header shows status dot badges and a summary line. Tap any header to collapse or expand.

SectionWhat it contains
🛢 OilEngine and generator oil change tracking. Cards show System — Component.
💧 Fluids & FiltersFuel filters, Racors, coolant, consumables. Cards show System — Component.
⚙️ ComponentsImpellers, belts, zincs, heat exchangers, everything else.

Section reordering — tap ⠿ Reorder to enter reorder mode. Drag handles appear; drag sections to reposition. Tap ✓ Done to exit. Order persists.

Component filter (Components section only) — pill row inside the Components section filters by a specific component. Oil and Fluids & Filters are unaffected.

Status color coding — 🟢 Green: within interval  ·  🟡 Amber: within 25% (due soon)  ·  🔴 Red: overdue. Both engine-hour and calendar-day intervals use the same color logic. Components without a configured interval show last service date and action only.

📋
Log
Full maintenance history, sorted newest first

Filter pills — tap Port Engine, Stbd Engine, Generator, Engines, or All to filter.

+ Add Entry — logs a new maintenance event.

FieldRequiredNotes
Date✅Format: M/D/YYYY — defaults to today
Engine Hours—Hours at time of service. Auto-hidden when the component+system combination has a date-based interval configured.
System✅Port Engine, Stbd Engine, Generator, or Engines
Action—Replaced, Checked, Repaired, Inspected
Component✅Dropdown from configured list — or type a new value
Part Number—e.g. WIX 33528
Notes—Free text
💡
Smart hours field: When you select a component+system that has a date-based interval configured (like a battery), the Engine Hours field auto-hides and shows a "Date-tracked" indicator. For hours-based components, the field appears as normal. Change either field to re-evaluate.
💡
Component field is a dropdown — pick from your configured list or type a new value. New values entered here are not automatically added to the list; manage the list in Settings → Components.

✎ Edit — tap the pencil icon on any row to open that entry prefilled. All fields are editable.

✕ Delete — tap the X icon on any row. Confirmation required.

⏱
Intervals
Configure and manage service intervals — drives all Overview status

Each row shows: component name, system, interval in hours, hours remaining or overdue status, last service date, and a progress bar.

+ Add Interval

FieldRequiredNotes
System✅Engine (all engine systems), individual system, or any configured system
Component✅Dropdown from configured list — must match log entry names
Interval Type✅Hours — engine hours between services (e.g. oil, filters)  ·  Days — calendar days (e.g. batteries, fire extinguishers, liferaft)
Interval (value)✅Number of hours or days between services
💡
Date-based intervals are useful for items that should be serviced on a calendar basis regardless of engine use: batteries (every 2 years), fire extinguishers (annual), liferaft repack (annual), bilge pump inspection (quarterly), etc. Status calculates from the date of the last log entry — engine hours field is optional for these.
⚠️
Component name matching: Names are case-insensitive but must otherwise match log entries. "Fuel Filter" matches "fuel filter" but not "Fuel Filters" (plural).

✎ Edit — opens the interval prefilled. Change system, component, or interval value.

✕ Delete — removes the tracking rule only. Log entries are unaffected.

Chart — compares hours since last service against the configured interval, color-coded by status.

Settings

Tap the Settings button in the upper right to open the Settings overlay.

SettingDescription
Boat NameUpdates header, browser title, and home screen label
Vessel IDUpdates the sub-header descriptor (e.g. OA 423)
Appearance🌙 Dark / ☀️ Light mode toggle — persists across sessions

Data

Data & Storage

All data is stored in Safari's local storage — persists across app closes, restarts, and reboots, but is tied to this device and file.

🔴
Data loss risk: Clearing Safari website data (Settings → Safari → Advanced → Website Data) resets the log to seed data. Export a backup regularly via Settings → Export Backup.

Storage keys

ss_log
Maintenance log entries
ss_intervals
Service intervals
ss_manual_hrs
Manual engine hour overrides
ss_config
Boat name, vessel ID, theme, section order
ss_systems
Configurable system list with colors
ss_components
Configurable component dropdown list
ss_data_version
Schema version for migrations

Upgrade workflow

Export Backup (Settings)
Replace MyDashboard.html on device
Re-add to Home Screen if needed
Open app — data loads untouched
Confirm version in Settings footer

If a schema change is needed in a future version, migrations run automatically. Use Import Backup if anything looks wrong after upgrading.

Network

Connectivity

Internet required on first load only, to fetch Google Fonts (Bebas Neue, DM Sans, DM Mono) and Chart.js. After Safari caches these, the app runs fully offline.

💡
For a completely offline build with all assets embedded in the file, ask Claude to inline them.
